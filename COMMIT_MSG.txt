## Technical Details

### Root Cause Fixed
Previous version only used launchctl load/unload which started daemon but didn't establish VPN connection. This caused:
- DNS not configured
- Network routes not set
- VPN tunnel not established

### Solutions Implemented
1. **PATH Fix**: Export full PATH and implement fallback warp-cli lookup
2. **Connection Management**: Call warp-cli connect/disconnect for actual VPN connection
3. **Error Handling**: Capture and display detailed error messages
4. **Conflict Detection**: Automatically detect and warn about network conflicts

## Real-World Issue Solved

User couldn't access company intranet (sg-git.pwtk.cc at 172.20.22.101) due to:
- Local Docker network using 172.20.0.0/16 (conflicting with company network)
- WARP daemon running but not establishing actual connection

Both issues are now detected and resolved with clear guidance.
